FCSys.Units

Constants and units of physical measure

Information

When a physical variable is assigned a quantity, it is the product of a number and a unit [BIPM2006]. In FCSys, units are also assigned numeric values in a consistent manner. A unit may be the product of any of the units defined in FCSys.Units raised to any power. When a quantity is converted to a number for display, it is divided by its unit. This conveniently handles unit conversion—without preference to any particular set of units. It can also be used to scale the floating point values associated with quantities. In order to provide well-scaled quantities, the value of the unit should be increased in inverse proportion to a typical value of a quantity expressed as a number in that unit. For example, if a model has a length scale on the order of a nanometer, it may be best to chose a unit system that results in a number of 109 for the unit meter.

Regardless of a quantity's unit, it is mapped through the relations in FCSys.Units (U) to a number times a product of five independent base constants or units each raised to the appropriate powers. These five are: the Rydberg constant (U.R_inf), speed of light in vacuum (U.c), Josephson constant (U.k_J), von Klitzing constant (U.R_K), and the candela (U.'cd'). The constants are related to units through the experimentally-derived definitions (e.g., U.Wb=483597.870e9/U.k_J). Here the designation of "constant" is in the physical sense; even the units are constant in the Modelica sense. These experimental relations, along with direct relations among the units (e.g., U.V=U.Wb*U.rad/U.s), establish the values of all the units of the International System of Units (SI) and other convenient units for modeling fuel cells.

This method of defining the base units from measured physical quantities (rather than vice versa) is natural and reflects the way that standards organizations define units. Here, the experimentally-derived relations for the physical constants are used rather the "exact" or "conventional" ones, where they exist [NIST2010].

There are five independent constants or units in FCSys.Units, but the International System of Units (SI) has seven independent base units (U.kg, U.m, U.s, U.A, U.mol, U.K, and U.'cd'). In FCSys, two additional constraints are imposed in order to simplify the model equations and allow electrons and chemical species to be to represented by the same base Species model. First, the Faraday constant (U.k_F or 96485.3399*U.C/U.mol) is normalized to one. This implies that the mole (U.mol) is proportional to the coulomb (U.C or U.Wb*U.S), which is considered a number of reference particles given the charge number. Also, the gas constant (U.R or 8.314472*U.J/(U.mol*U.K)) is normalized to one. Therefore, the kelvin (U.K) is proportional to the volt (U.V or U.J/U.C). In addition, the radian U.rad is defined as a base constant. However, it must be set equal to one in the current version International System of Units (SI) [BIPM2006].

The user should be aware of several aspects of the implementation. A script ("FCSys/resources/scripts/units.mos") runs a model (Units.Evaluate) to determine the values of the units and defines the conversions to display values as numbers in units. This script is set to run when FCSys is loaded via "FCSys/load.mos" or from a link in the "Commands" menu of FCSys.Units (when viewed within Dymola). Unless the values of the base constants and units are changed, it should be sufficient to simply allow the script to run when the package is loaded. When the der() operator is utilized, it must be divided by the unit second (U.s), since this scaling is not automatic in Modelica's specification and Dymola's implementation. Currently, it is not possible to define a function that implements a der() function with scaling.

Note that the common "natural" units systems (Planck, Stoney, Hartree, Rydberg, or Natural) cannot be implemented given the constraints that the gas and Faraday constants equal one. Since these systems set the Boltzmann constant equal to one, the Planck constant must equal the von Klitzing constant. That is not the case for any of these systems of units (http://en.wikipedia.org/wiki/Natural_units). The structure of FCSys.Units allows the constraints on the Faraday and gas constants to be relaxed, but the models in FCSys generally do not.

Although it is not necessary in Modelica, the unit declarations are presorted so that they can be easily ported to imperative/causal languages (e.g., Python, C).

For more information, see the paper on natural units [Davies and Paredis, 2012].

FCSys.Units.Evaluate

Evaluate the values assigned to constants and units

Information

This model is used by the units script ("FCSys/resources/scripts/units.mos") to establish the values of the units in order to convert values to numbers for display.

For more information, see the documentation in the FCSys.Units package.
