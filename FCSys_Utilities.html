FCSys.Utilities.arrayIntegerEqual

Check if two arrays of integers are equal

Information

Examples:
arrayIntegerEqual({1,2}, {1,2}) returns true, but arrayIntegerEqual({1,2}, {1,3}) and arrayIntegerEqual({1,2}, {1,2,3}) each return false.

Extends from Modelica.Icons.Function (Icon for functions).

Inputs

TypeNameDefaultDescription
Integeru1[:] First integer vector
Integeru2[:] Second integer vector

Outputs

TypeNameDescription
Booleanequaltrue, if all of the entries are equal

Modelica definition

function arrayIntegerEqual 
  "Check if two arrays of integers are equal"
  extends Modelica.Icons.Function;

  input Integer u1[:] "First integer vector";
  input Integer u2[:] "Second integer vector";
  output Boolean equal 
    "true, if all of the entries are equal";

algorithm 
  if size(u1, 1) <> size(u2, 1) then
    equal := false;
    return;
  end if;
  for i in 1:size(u1, 1) loop
    if u1[i] <> u2[i] then
      equal := false;
      return;
    end if;
  end for;
  equal := true;
  return;
end arrayIntegerEqual;

FCSys.Utilities.arrayRealEqual

Check if two arrays of Real variables are essentially equal

Information

Examples:
arrayRealEqual({1,2}, {1,2}) returns true, but arrayRealEqual({1,2}, {1,2.001}) and arrayRealEqual({1,2}, {1,2,3}) each return false.

Realu2[:] Second real vector

Outputs

Modelica definition

function arrayRealEqual 
  "Check if two arrays of Real variables are essentially equal"
  extends Modelica.Icons.Function;

  input Real u1[:] "First real vector";
  input Real u2[:] "Second real vector";
  input Real eps=1e-7 "Error tolerance";
  output Boolean equal 
    "true, if all of the entries are equal";

algorithm 
  if size(u1, 1) <> size(u2, 1) then
    equal := false;
    return;
  end if;
  for i in 1:size(u1, 1) loop
    if abs(u1[i] - u2[i]) > eps then
      equal := false;
      return;
    end if;
  end for;
  equal := true;
  return;
end arrayRealEqual;

FCSys.Utilities.assertEval

Assert function that forces Dymola to parse the message

Information

When an assert statement is false in the initial equation section of a model, Dymola 7.4 gives the following error during translation: 

"Error: The conditions of the following assert statements are always false:"
without parsing the message given to the assert function. This pass-through function causes the statement to be evaluated during initialization, at which point the message is evaluated.

Extends from Modelica.Icons.Function (Icon for functions).

Inputs

TypeNameDefaultDescription
Booleancondition  
Stringmessage  

Modelica definition

function assertEval 
  "Assert function that forces Dymola to parse the message"
  extends Modelica.Icons.Function;
  input Boolean condition;
  input String message;

algorithm 
  assert(condition, "\n" + message + "\n");
end assertEval;

  extends Modelica.Icons.Function;

FCSys.Utilities.enumerate

Enumerate the true entries in a Boolean vector (0 for false entries)

Information

Example:
enumerate({true,false,true}) returns {1,0,2}.

Extends from Modelica.Icons.Function (Icon for functions).

Inputs

TypeNameDefaultDescription
Booleanu[:] Boolean vector

Outputs

TypeNameDescription
Integerenumerated[size(u, 1)]Indices of the true entries (increasing order; 0 for false entries)

Modelica definition

function enumerate 
  "Enumerate the true entries in a Boolean vector (0 for false entries)"
  extends Modelica.Icons.Function;
  input Boolean u[:] "Boolean vector";
  output Integer enumerated[size(u, 1)] 
    "Indices of the true entries (increasing order; 0 for false entries)";

protected 
  Integer count "Counter variable";

algorithm 
  count := 1;
  for i in 1:size(u, 1) loop
    if u[i] then
      enumerated[i] := count;
      count := count + 1;
    else
      enumerated[i] := 0;
    end if;
  end for;
end enumerate;

  extends Modelica.Icons.Function;
